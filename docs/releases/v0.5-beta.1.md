# v0.5-beta.1

Release date: 2026-05-17
Target: `main` at tag `v0.5-beta.1`
Stability: Beta prerelease, not production-ready

## Summary

This beta release improves the public demo chrome, CommonMark HTML handling,
and Typora custom theme compatibility. It keeps the project native and
framework-free while adding bilingual website UI, sanitized HTML block
rendering, and a stricter custom-theme normalization path for dark Typora CSS
files.

## Features

- Added English/Chinese language switching to the website navigation.
- Added i18n coverage for website chrome, editor toolbar labels, status
  messages, specs page copy, and spec card controls while leaving editor
  document content untouched.
- Added CommonMark HTML block parsing as a dedicated ProseMirror block node.
- Added DOMPurify-based sanitized rendering for CommonMark HTML blocks.
- Added original-source serialization for parsed HTML blocks so save/export
  keeps the authored HTML.

## Fixes

- Added pointer cursor styling for links inside editable article content.
- Improved custom Typora theme selector normalization for `html`, `body`,
  `:root`, `#write`, `.md-fences`, math, TOC, focus, and task-list selectors.
- Mirrored document-level theme background and color rules onto the editor
  wrapper, ProseMirror surface, and source textarea so self-contained dark
  themes visibly affect the editing background.
- Replaced fragile regex-based CSS rewriting with a small block scanner that
  preserves nested `@media`, `@supports`, `@container`, and `@keyframes`
  blocks.
- Updated autolink negative coverage so it does not conflict with valid
  CommonMark HTML block syntax.

## Documentation

- Updated README coverage to describe the CommonMark 0.31.2 baseline.
- Documented sanitized HTML block support and the remaining inline HTML
  limitation.
- Marked imported Typora CSS themes as experimental and documented browser
  limitations around multi-file themes that rely on relative `@import` or
  `url(...)` references.
- Removed the obsolete local Grok planning note from the working tree.

## Dependency Changes

- Added `dompurify` for browser-side HTML sanitization.

## Verification

- `pnpm test` passed.
- `pnpm build` passed.
- `pnpm build:lib` passed.
- `git diff --check` passed.
- Browser smoke testing covered language switching, editor link cursor style,
  sanitized HTML block rendering, and dark Typora theme background import.

## Known Limitations

- This is a beta prerelease and is not production-ready.
- Custom Typora CSS theme import remains experimental. Single-file themes are
  the supported path; multi-file themes with relative imports may need to be
  flattened before import.
- Inline HTML is preserved as source text but is not yet rendered as sanitized
  inline DOM.
- Unknown or unsafe HTML removed by DOMPurify may not visually match Typora's
  permissive raw HTML rendering.
