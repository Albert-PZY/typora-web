# v0.6-beta.1

## Overview

This beta release tightens the default editing experience around built-in styling and code-block rendering. It replaces the experimental external Typora CSS import path with a predictable built-in light/dark appearance model, and fenced code blocks now use CodeMirror 5 runmode tokenization for Typora-aligned syntax highlighting.

This release is still a beta and is not recommended for production use.

## What's Changed

- Added CodeMirror 5 as the code-block highlighting engine.
- Added a `src/code-highlighter.ts` adapter that maps supported fence languages to CodeMirror modes and emits official `cm-*` token classes.
- Added ProseMirror inline decorations for code-block tokens so highlighted code remains editable inside the existing `code_block` NodeView.
- Added built-in light and dark appearances controlled by `data-appearance` on the document root.
- Added a top-right appearance toggle beside the GitHub link, with English and Chinese labels.
- Extended dark styling across the website shell, editor paper, source textarea, editor widgets, table controls, Mermaid panels, math surfaces, and CodeMirror token colors.
- Removed runtime custom Typora CSS theme import from the public editor controller.
- Removed custom theme import and clear controls from the demo toolbar.
- Updated README and syntax survey documentation to describe built-in appearances and CodeMirror highlighting.
- Added repository memory notes for the new highlighting and appearance contracts.

## Removed

- Removed `src/theme.ts`.
- Removed `editor.importThemeFile(file)`.
- Removed `editor.applyThemeCss(name, cssText)`.
- Removed `editor.clearCustomTheme()`.
- Removed `editor.getCustomThemeName()`.
- Removed the demo toolbar's external CSS theme file picker.

## Tests

- Added focused tests for CodeMirror token output and DOM rendering inside fenced code blocks.
- Added focused tests for built-in appearance persistence, root attributes, localized navigation text, and toggling.
- Replaced custom theme import tests with negative API/UI tests proving the external CSS theme import surface is gone.
- Verified full test suite, website production build, library build, diff whitespace checks, and browser smoke behavior for dark appearance, CodeMirror token rendering, and Mermaid source hiding after successful render.

## Compatibility Notes

- External Typora `.css` themes are no longer supported at runtime. Use the bundled default styles and built-in light/dark appearance switching.
- Unsupported fenced-code languages remain plain instead of using guessed tokenization.
- Version tags continue to use the beta format `vx.x-beta.x`; production tags should use `vx.x.x` only after a manual production-readiness decision.
