# v0.7-beta.2

Release date: 2026-05-17
Target: main @ v0.7-beta.2
Stability: Beta prerelease, not production-ready

## Summary

This beta release focuses on editor correctness and polish after the
CodeMirror 6 milestone. It fixes inline mark position drift after task markers,
prevents source-mode shortcut collisions, refines click-to-source behavior for
rich Markdown blocks, restores sanitized HTML block previews, improves the
fenced-code language picker, and completes bilingual project documentation.

## Fixes

- Fixed task-list inline mark alignment so inline code, strong text, and other
  Method-B marks no longer swallow or shift adjacent characters after checkbox
  markers.
- Fixed Markdown serialization around inline atom nodes so task markers do not
  incorrectly advance text-based serializer offsets.
- Fixed repeated `Ctrl+/` presses in source mode so the global source toggle
  wins before CodeMirror line-comment handling can mutate Markdown content.
- Fixed Mermaid source reveal behavior: successful diagrams hide their source
  frame by default, render below the source frame instead of inside it, reveal
  source when clicked, and hide source again when the user clicks elsewhere.
- Fixed KaTeX block math reveal behavior so rendered formulas can reveal source
  on click and hide it again on outside click while keeping invalid formulas
  visible for editing.
- Fixed CommonMark HTML block preview behavior so sanitized HTML renders as
  normal content by default, source is revealed on click, and native
  `<details>` / `<summary>` disclosure controls are normalized out of the
  preview.
- Fixed the fenced-code language selector by replacing the native datalist with
  a scrollable custom menu that displays only language names and respects dark
  appearance styles.
- Fixed code block wrapping so embedded CodeMirror editors, source mode, and
  the default Markdown themes wrap long code lines instead of forcing
  horizontal scrolling.
- Fixed website logo and favicon loading in subpath deployments by serving the
  asset from the configured public directory and using relative paths.
